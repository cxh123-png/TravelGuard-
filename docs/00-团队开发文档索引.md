# TravelGuard 团队开发文档索引

## 权威来源与阅读顺序

1. 根目录 [`实现方案.md`](../实现方案.md)：业务、架构、状态机、技术栈与全阶段验收的唯一权威来源。
2. [整体实施路线图](01-整体实施路线图.md)：七个模块的依赖关系、阶段门槛和合并顺序。
3. [协作、PR 与验收规范](02-协作PR与验收规范.md)：分支创建、AI 使用边界、PR 审阅与合并标准。
4. [任务包索引](tasks/README.md)：每名成员的可执行任务包。

## 使用方式

每位成员开始任何工作前必须：

1. 拉取最新 `main`；
2. 阅读本索引、整体路线图、协作规范和自己的任务包；
3. 检查任务包的“开始条件”是否已满足；
4. 从 `main` 新建功能分支；
5. 仅让 AI 修改任务包允许的目录；
6. 按任务包运行测试、提交 PR 并附上验收证据。

成员不得把“前置任务尚未合并”当成自行修改其他模块的理由。需要接口或 Schema 时，在对应 Issue/PR 中提出请求，由相关模块补齐或先合并契约 PR。

## 当前文档状态

- 本文档仅定义团队执行方法，不代表任何功能已经实现。
- 本仓库的 Python 运行时固定为 Python 3.11；依赖见根目录 `requirements.txt`。
- 前端只是展示和人工操作界面，不能复制政策、预算、审批或合规判断逻辑。
